# Curriculum Format

What a well-formed curriculum document looks like.

## Structure

```markdown
# [Course Name] -- Curriculum

## Module 1: [Module Title]

### Session 1.1: [Session Title]
- **Duration:** [X] minutes
- **Learning objectives:**
  1. [Learner will be able to...]
  2. [Learner will be able to...]
- **Content:**
  - [Chunk reference or summary]
  - [Chunk reference or summary]
- **Key takeaway:** [One sentence the learner should remember]

### Session 1.2: [Session Title]
...

## Module 2: [Module Title]
...
```

## Rules

1. Each session should cover 3-7 content chunks (not more)
2. Each session needs at least 2 learning objectives
3. Learning objectives start with action verbs: explain, identify, compare, apply, build, evaluate
4. Sessions within a module build on each other -- order matters
5. The first session of each module should be accessible without the previous module (for flexible delivery)
6. Flag any session that requires charts, data visualization, or images -- this informs Stage 03 outline

## Session Sizing

| Deck Length | Content Density | Recommended For |
|-------------|----------------|-----------------|
| 10-15 slides | Light -- overview or intro | Short sessions, kickoffs |
| 15-25 slides | Standard -- most sessions | Core teaching content |
| 25-35 slides | Dense -- deep dives | Technical or data-heavy sessions |

Target the standard range unless the content clearly demands otherwise.
